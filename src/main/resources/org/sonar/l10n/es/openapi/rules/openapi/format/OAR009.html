Al definir una API se deben definir cuáles son los formatos de entrada de datos.

Según el estándar el formato por defecto será application/json, salvo algunas excepciones.

Ejemplo de código no compatible (OpenAPI 2)

swagger: "2.0"
info:
  title: Swagger Petstore
  version: "1.0"
consumes:
  - application/xml
paths:
  /pets:
    put:
      responses: 
        200:
          description: OK

Solución compatible (OpenAPI 2)

swagger: "2.0"
info:
  title: Swagger Petstore
  version: "1.0"
consumes:
  - application/json
paths:
  /pets:
    put:
      responses: 
        200:
          description: OK

Ejemplo de código no compatible (OpenAPI 3)

openapi: 3.0.0
info:
  title: Swagger Petstore
  version: "1.0"
paths:
  /pets:
    post:
      requestBody:
        description: Optional description in *Markdown*
        required: true
        content:
          application/xml:
            schema:
              type: object
      responses:
        200:
          description: Ok

Solución compatible (OpenAPI 3)

openapi: 3.0.0
info:
  title: Swagger Petstore
  version: "1.0"
paths:
  /pets:
    post:
      requestBody:
        description: Optional description in *Markdown*
        required: true
        content:
          application/json:
            schema:
              type: object
      responses:
        200:
          description: Ok
