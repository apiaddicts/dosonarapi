Todos los parámetros en query deben definirse como opcionales

Ejemplo de código no compatible (OpenAPI 2)

  swagger: "2.0"
  info:
    title: Swagger Petstore
    version: 1.0.0
  paths:
    /pets:
      get:
        parameters:
          - in: query
            name: $expand
            type: array
            required: true
            items:
              type: string
        responses:
          200:
            description: Ok

Solución compatible (OpenAPI 2)

swagger: "2.0"
info:
  title: Swagger Petstore
  version: 1.0.0
paths:
  /pets:
    get:
      parameters:
        - in: query
          name: $expand
          type: array
          required: false
          items:
            type: string
      responses:
        200:
          description: Ok
