Los parámetros query en las operaciones GET deben definirse de acuerdo con el estándar

Ejemplo de código no compatible (OpenAPI 2)

En el caso que param1, param2 and param3 sean obligatorios

  swagger: '2.0'
  info:
    version: 1.0.0
    title: My API
  paths:
    /items:
      get:
        summary: Get a list of items
        parameters:
          - name: param1
            in: query
            type: string
          - name: param3
            in: query
            type: boolean
          - name: param4
            in: query
            type: string
          - name: param5
            in: query
            type: number
        responses:
          200:
            description: A list of items
            schema:
              type: array
              items:
                type: string

Solución compatible (OpenAPI 2)

  swagger: '2.0'
  info:
    version: 1.0.0
    title: My API
