Defina la propiedad summary para cada operación. La especificación OpenAPI es una forma de transmitir significado a usuarios no técnicos. El resumen ayudará a comprender de qué se trata la operación, en lugar de abordarla en términos técnicos.

Ejemplo de código no compatible (OpenAPI 2)

swagger: "2.0"
info:
  version: 1.0.0
  title: Swagger Petstore
paths:
  /pets:
    post:
      responses:
        default:
          description: the default respons
    get:
      responses:
        default:
          description: the default response

Solución compatible (OpenAPI 2)

swagger: "2.0"
info:
  version: 1.0.0
  title: Swagger Petstore
paths:
  /pets:
    post:
      summary: Create a new pet
      responses:
        default:
          description: the default response
    get:
      summary: List all pets
      responses:
        default:
          description: the default response
