Al definir una API se deben definir cuáles son los formatos de entrada de datos.

Para OpenAPI 2 especificar al menos un tipo MIME comprendido por la API en la palabra clave "consumes" es obligatorio

Para OpenAPI 3 especificar al menos un tipo de medio en el contenido del cuerpo de la solicitud es obligatorio

Ejemplo de código no compatible (OpenAPI 2)

swagger: "2.0"
info:
  title: Swagger Petstore
  version: "1.0"
paths:
  /pets:
    put:
      responses: 
        200:
          description: OK

Solución compatible (OpenAPI 2)

swagger: "2.0"
info:
  title: Swagger Petstore
  version: "1.0"
consumes:
  - application/json
paths:
  /pets:
    put:
      responses: 
        200:
          description: OK

Or

swagger: "2.0"
info:
  title: Swagger Petstore
  version: "1.0"
paths:
  /pets:
    put:
      consumes:
        - application/json
      responses: 
        200:
          description: OK

Ejemplo de código no compatible (OpenAPI 3)

openapi: 3.0.0
info:
  title: Swagger Petstore
  version: "1.0"
paths:
  /pets:
    post:
      responses:
        200:
          description: Ok

Solución compatible (OpenAPI 3)

openapi: 3.0.0
info:
  title: Swagger Petstore
  version: "1.0"
paths:
  /pets:
    post:
      requestBody:
        description: Optional description in *Markdown*
        required: true
        content:
          application/json:
            schema:
              type: object
      responses:
        200:
          description: Ok
