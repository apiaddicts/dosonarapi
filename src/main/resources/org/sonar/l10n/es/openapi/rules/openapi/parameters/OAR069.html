Todos los parámetros en query deben definirse como opcionales

Ejemplo de código no compatible (OpenAPI 2)

  swagger: '2.0'
info:
  version: 1.0.0
  title: Swagger Petstore
paths:
  /items:
    get:
      summary: Get a list of items
      parameters:
        - name: param1
          in: query
          required: false
          type: string
      responses:
        200:
          description: A list of items
          schema:
            type: array
            items:
              type: string

Solución compatible (OpenAPI 2)

  swagger: '2.0'
  info:
    version: 1.0.0
    title: Swagger Petstore
  paths:
    /items:
      get:
        summary: Get a list of items
        parameters:
          - name: param1
            in: query
            required: false
            type: string
        responses:
          200:
            description: A list of items
            schema:
              type: array
              items:
                type: string
          400:
            description: Bad Request
