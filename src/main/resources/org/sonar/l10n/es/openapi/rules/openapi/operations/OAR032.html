Al leer una URL, debe ser posible interpretar a qué recursos se está refiriendo de manera concreta.

Deben evitarse los nombres ambiguos en los recursos. La recomendación es utilizar nombres específicos.

Ejemplo de código no compatible (OpenAPI 2)

swagger: "2.0"
info:
  title: Swagger Petstore
  version: 1.0.0
paths:
  /invoices/{id}/elements:
    get:
      parameters:
        - name: id
          in: path
          required: true
          type: integer
      responses:
        200:
          description: Ok

Solución compatible (OpenAPI 2)

swagger: "2.0"
info:
  title: Swagger Petstore
  version: 1.0.0
paths:
  /invoices/{id}/products:
    get:
      parameters:
        - name: id
          in: path
          required: true
          type: integer
      responses:
        200:
          description: Ok
