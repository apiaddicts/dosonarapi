Definir la respuesta "default" es obligatorio.

La respuesta "default" se utiliza para todos los códigos HTTP que no se tratan individualmente.

Ejemplo de código no compatible (OpenAPI 2)

swagger: "2.0"
info:
  title: Swagger Petstore
  version: "1.0"
paths:
  /pets:
    get:
      responses:
        200:
          description: OK

Solución compatible (OpenAPI 2)

swagger: "2.0"
info:
  title: Swagger Petstore
  version: "1.0"
paths:
  /pets:
    get:
      responses:
        200:
          description: OK
        default:
          description: Error
