Al definir una API se deben definir cuáles son los formatos de salida de datos.

Para OpenAPI 2 especificar al menos un tipo MIME comprendido por la API en la palabra clave "produces" es obligatorio.

Para OpenAPI 3 especificar al menos un tipo de medio en el contenido del cuerpo de la respuesta es obligatorio.

Ejemplo de código no compatible (OpenAPI 2)

swagger: "2.0"
info:
  title: Swagger Petstore
  version: "1.0"
paths:
  /pets:
    put:
      responses: 
        200:
          description: OK

Solución compatible (OpenAPI 2)

swagger: "2.0"
info:
  title: Swagger Petstore
  version: "1.0"
produces:
  - application/json
paths:
  /pets:
    put:
      responses: 
        200:
          description: OK

Or

swagger: "2.0"
info:
  title: Swagger Petstore
  version: "1.0"
paths:
  /pets:
    put:
      produces:
        - application/json
      responses: 
        200:
          description: OK

Ejemplo de código no compatible (OpenAPI 3)

openapi: 3.0.0
info:
  title: Swagger Petstore
  version: "1.0"
paths:
  /pets:
    post:
      responses:
        200:
          description: Ok

Solución compatible (OpenAPI 3)

openapi: 3.0.0
info:
  title: Swagger Petstore
  version: "1.0"
paths:
  /pets:
    post:
      responses:
        200:
          description: Ok
          content:
            application/json:
              schema:
                type: object
